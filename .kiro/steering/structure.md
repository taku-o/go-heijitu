## ディレクトリパターン

### Go プロジェクト標準構成（想定）
**ルートパッケージ**: `/`
- エントリーポイント（`main.go` など）

**内部パッケージ**: `/internal/`
- 外部公開しない内部ロジック

**公開パッケージ**: `/pkg/` （必要な場合）
- 外部から利用可能なパッケージ

**コマンド**: `/cmd/`
- 複数バイナリがある場合の各エントリーポイント

## 命名規則

- **ファイル**: snake_case（Go 標準）
- **パッケージ**: lowercase, 短い名前
- **関数/メソッド**: CamelCase（公開）/ camelCase（非公開）
- **定数**: CamelCase（公開）/ 全大文字は避ける

## インポート整理

```go
import (
    // 標準ライブラリ
    "fmt"
    "os"

    // 外部ライブラリ
    "github.com/some/library"

    // 内部パッケージ
    "github.com/taku-o/go-heijitu/internal/something"
)
```

## コード構成の原則

- パッケージは単一の責務を持つ
- 循環インポートを避ける
- インターフェースは使用側で定義する（Go の慣習）
